# Initialize Superpowers Lite Configuration

Generate a `.superpowers.yml` config file for the current project.

## Process

1. **Check if config already exists**
   - If `.superpowers.yml` exists, ask: "Config already exists. Overwrite, edit, or cancel?"

2. **Ask preference questions (one at a time):**

   **Question 1: Workflow mode**
   > How should I handle task routing?
   > a) **Auto** — I decide based on task size (recommended)
   > b) **Full** — Always use the full pipeline (brainstorming → spec → plan → execute)
   > c) **Quick** — Always fast-track (for maintenance/iteration work)

   **Question 2: TDD strictness**
   > How strict should test-driven development be?
   > a) **Required** — No code without a failing test first (strict TDD)
   > b) **Recommended** — TDD encouraged but not enforced
   > c) **Off** — No TDD requirement

   **Question 3: Review mode**
   > How should code reviews work?
   > a) **Separate** — Spec review then quality review (most thorough)
   > b) **Combined** — Single review pass (faster, fewer tokens)
   > c) **Off** — Skip reviews entirely

   **Question 4: Auto-commit**
   > Should I commit automatically after each task?
   > a) **Yes** — Auto-commit after each completed task
   > b) **No** — I'll tell you when to commit

   **Question 5: Document paths**
   > Where should I save design specs and implementation plans?
   > a) **Default** — `docs/specs/` and `docs/plans/`
   > b) **Custom** — Let me specify paths

   **Question 6: Preset (optional)**
   > Want to start from a preset?
   > a) **None** — Use the answers above
   > b) **Startup** — Fast iteration, minimal reviews
   > c) **Enterprise** — Full pipeline, strict TDD, all reviews
   > d) **Learning** — Extra explanations and checkpoints

3. **Generate the config file** based on answers

4. **Show the generated config** and ask for confirmation

5. **Write to `.superpowers.yml`** and add `.superpowers/` to `.gitignore` if not already there

## Example Output

```yaml
# Generated by superpowers-lite init
mode: auto
tdd: recommended
auto_commit: true
review_mode: combined
paths:
  specs: "docs/specs"
  plans: "docs/plans"
```
